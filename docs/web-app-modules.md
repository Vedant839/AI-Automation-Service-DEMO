# Web App Modules

The web app should be an operations console. Each screen should help a real staff member do work faster or safer.

## Admin Dashboard

Purpose:

- Give management a daily overview of automation value and risk.

MVP:

- Student activity analytics
- Risk alert counts
- Escalation monitoring
- AI usage summary
- Failed automation jobs
- Campaign status

Later:

- Counselor performance trends
- Retention impact
- Cost analysis
- Custom dashboards by department

## Teacher Portal

Purpose:

- Help teachers understand where students are struggling.

MVP:

- Weak topics
- Common doubts
- Flagged students
- Daily AI summary
- Evidence links to doubts, quizzes, or attendance events

Later:

- Topic-level drilldown
- Lesson recommendations
- Batch comparison
- Teacher notes and feedback loop

## Counselor Workspace

Purpose:

- Turn AI alerts into human follow-up.

MVP:

- Assigned student queue
- Priority and reason
- Conversation history
- Parent message preview
- Mark contacted/resolved

Later:

- Take over live AI chats
- Approve outbound campaigns
- SLA timers
- Outcome tracking
- Call notes and reminders

## Knowledge Base Manager

Purpose:

- Keep AI answers grounded in approved client content.

MVP:

- Upload PDFs, schedules, policies, notes, and fee docs
- Show indexing status
- Mark content approved/unapproved
- View source snippets used by AI

Later:

- Version comparison
- Approval workflow
- Scheduled re-indexing
- Content quality score

## AI Monitoring Panel

Purpose:

- Make production AI behavior observable and correctable.

MVP:

- Failed responses
- API failures
- Latency
- Token usage
- Escalation rate
- Low-confidence responses
- Hallucination reports

Later:

- Model comparison
- Cost per workflow
- Retrieval hit quality
- Prompt version analytics
- Regression testing dashboard
